      Como posso redimensionar uma instância existente?

      Embora a opção Redimensionar instância apareça no Horizon, essa funcionalidade atualmente não é suportada. Se for necessário ampliar o tamanho de uma instância, o caminho recomendado é fazer uma captura instantânea da instância e reimplementar essa captura instantânea em um tipo novo, maior.

      Como posso inicializar uma instância em um host específico?

      Esta função é suportada por meio do OpenStack API. Consulte Como implementar uma instância em um nó do hypervisor específico para obter mais detalhes.

      Eu tenho uma nuvem nova e já existem dois projetos em vigor. Quais são eles?

      Os projetos admin e service são criados para suportar a operação subjacente do seu IBM Bluemix Private Cloud. Esses projetos devem continuar em vigor. Não os modifique nem implemente.

      Quais portas/URLs etc são publicamente acessíveis no meu Bluemix Private Cloud e para qual propósito?

      As portas que são abertas por meio do firewall para o Bluemix Private Cloud são:

      80 - horizon (OpenStack Web Portal)
443 - horizon (OpenStack Web Portal)
5000 - keystone (OpenStack Authentication API)
6080 - nova-novnc (OpenStack VNC Console)
8000 - heat (OpenStack CloudFormation API)
8004 - heat (OpenStack Orchestration API)
8774 - nova (OpenStack Compute API)
8776 - cinder (OpenStack Block Storage API)
9292 - glance (OpenStack Image API)
9696 - neutron (OpenStack Network API)
35357 - keystone-admin (OpenStack Authentication Admin API)

      Três portas adicionais serão ativadas se você tiver solicitado que o Swift (Armazenamento de Objeto do OpenStack) seja configurado em seu cluster: 6000, 6001, 6002

      Todas essas portas estão disponíveis por meio de sua URL do Horizon customizada.

      Se também for necessário restringir as portas de egresso para o mínimo necessário, você precisará abrir pelo menos as portas 67/68 para DHCP e a porta 80 para metadados.

      Como posso medir Operações de ES Por Segundo em uma unidade da instância ou do volume?

      Você pode usar qualquer número de ferramentas para medir o desempenho do disco.

      Um script Python particularmente fácil de usar para medir IOPS de Benjamin Schweizer com instruções e resultados de exemplo está disponível:

      https://benjamin-schweizer.de/measuring-disk-io-performance.html

      A versão mais recente deste script atualmente está disponível aqui.

      O uso é o seguinte:

      iops [-n|--num_threads threads][-t|--time time] <device>

      Parâmetros:

      threads := number of concurrent io threads, default 1
time    := time in seconds, default 10
device  := some block device, like /dev/sda or \\\\.\\PhysicalDrive0

      Exemplos:

      iops /dev/sda
iops -n 8 -t 2 /dev/disk0
iops --num_threads 1 --time 2 /dev/md1
iops --num_threads 16 --time 2 /dev/md1

      Ao usar grupos de segurança, como posso registrar conexões TCP recusadas para portas proibidas?

      Se desejar registrar conexões TCP recusadas para portas proibidas, você precisará abrir grupos de segurança e executar iptables com criação de log nas instâncias.

      Qual é o número máximo de dispositivos virtuais (por exemplo, portas) que podem ser anexados a uma máquina virtual?

      20 é o número máximo de slots PCI virtuais disponíveis.

      Como posso obter uma lista de todas as minhas MVs e seus nomes de host?

      Precisa de uma lista de todas as instâncias em uma nuvem com o host no qual elas residem? Precisa dos nomes das MVs também? Se você for cloud_admin, há um liner para isso!

      nova list --all-tenants --fields name,OS-EXT-SRV-ATTR:host

      O comando openstack host show disk name mostra o uso real de disco rígido do host físico?

      Não. Ele mostra informações sobre os tipos das instâncias que são executadas nos hosts. Nesse momento, informações de uso de troca não são incluídas. A seção a seguir mostra o que está incluído a partir da liberação Mitaka.

      Consulte também https://derpops.bike/2016/10/07/openstack-nova-and-hypervisor-disk-consumption/

      Para ajudar a responder essa pergunta, precisamos voltar para a documentação do OpenStack: http://docs.openstack.org/admin-guide/common/nova-show-usage-statistics-for-hosts-instances.html

      • A coluna disk_gb [na saída openstack host show] mostra a soma dos tamanhos de disco raiz e efêmero (em GB) das instâncias que são executadas no host. [Este valor é] calculado utilizando informações sobre os tipos das instâncias executadas nos hosts. Este comando não consulta o … uso de disco rígido do host físico.
      • É possível ver o cálculo de tipo real em https://github.com/openstack/nova/blob/c521ba7fa679b6e9c790d84ea22f04fb999987db/nova/notifications/base.py#L448-L454 disk_gb = root_gb + ephemeral_gb
      • ephemeral_gb = Especifica o tamanho de um disco de dados efêmero secundário. Este é um disco vazio, não formatado que existe apenas enquanto durar a instância. O valor padrão é 0.
      • root_gb = Quantidade de espaço em disco (em gigabytes) a ser usado para a partição raiz (/).
      • Atualmente, o disk_gb do Nova NÃO inclui informações de troca. (Ou seja, alocação de espaço de troca opcional para a instância. O valor padrão é 0.)

      O trabalho é proposto para a liberação do OpenStack Newton que seria iniciado incluindo esse espaço de troca opcional, a partir desta especificação https://specs.openstack.org/openstack/nova-specs/specs/newton/implemented/resource-providers-allocations.html:

      “Quando o nó de cálculo utiliza armazenamento local para discos da instância OU foi inicializado a partir do volume, o valor usado deve ser a soma dos valores dos campos root_gb, ephemeral_gb e swap do tipo. O resource_provider_uuid deve ser o UUID do nó de cálculo. Observe que, para as instâncias que foram inicializadas a partir do volume, o valor de root_gb será 0.”

      Como criar um volume a partir de uma captura instantânea e remover dependências?

      Ao criar um volume a partir de uma captura instantânea, a captura instantânea original pode conter (meta)dados dependentes necessários para os volumes. É possível executar openstack volume show $volumeid para localizar o snapshot_id pai. Ao tentar excluir o volume, pode ocorrer um erro devido à necessidade de a captura instantânea dependente ser excluída primeiro:

      Falha ao excluir a captura instantânea com o nome ou ID...

      Uma alternativa para isso é criar uma imagem a partir do volume e, em seguida, criar um volume a partir dessa imagem:

      Etapa 1: criar uma imagem temporária a partir de um volume:

      openstack image create --volume $volumeid $image_name

      Etapa 2: criar um volume a partir da imagem temporária:

      openstack volume create --image $imageID --size $x

      É possível então excluir a imagem temporária e você terá um volume independente.
