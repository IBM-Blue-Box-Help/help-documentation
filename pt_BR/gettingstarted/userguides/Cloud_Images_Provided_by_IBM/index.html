      User's Guide to Cloud Images (Community Edition)

      • by Yanqi Dou ● March 9th, 2016

      • Tags:

        • userguides

        • images

        • glance

        • image-create

        • instance

      O IBM Bluemix Private Cloud 3.0 é fornecido previamente preenchido com imagens x86_64 do Cirros 0.3.3. O IBM Bluemix Private Cloud fornece outras imagens de nuvem que podem ser transferidas por download, para que seja possível executar as instâncias com o Sistema operacional de sua escolha. Este documento informa o que é preciso saber para fazer download e trabalhar com essas imagens. Para certificar-se de estar trabalhando com as imagens mais recentes, é possível consultar as Notas sobre a liberação de imagens da nuvem a qualquer momento.

      Informações adicionais sobre como usar imagens com o OpenStack estão disponíveis no website do OpenStack: https://docs.openstack.org/image-guide/

      Pré-requisitos

      1. Caso planeje usar ferramentas da linha de comandos (CLI), certifique-se de que o OpenStack Glance Client esteja instalado. É possível encontrar instruções de instalação na base de conhecimento: Introdução à API do OpenStack.

      2. Cerifique-se de ter acesso ao Box Panel, para que seja possível obter as URLs de download provisórias para as imagens de nuvem fornecidas pela IBM.

      3. O upload de imagens para o Glance utilizando tempUrl como um local de imagem é suportado somente no Bluemix Private Cloud 2.1.0 em diante.

      Conceitos Básicos

      Item Descrição
      **Imagem  
      ** Uma imagem de máquina virtual, chamada neste documento simplesmente de imagem, é um único arquivo que contém um disco virtual com um sistema operacional inicializável instalado nele. Use imagens para criar instâncias de máquina virtual dentro da nuvem.
      **Instância  
      ** Instâncias são máquinas virtuais (MVs) executadas dentro da nuvem do OpenStack.
      **Tipo ** O termo tipo descreve o tamanho de uma instância e define as especificações de recurso predeterminadas para a quantidade de vCPU, de RAM e de espaço em disco disponível para uma instância.

      Selecione a partir das imagens de nuvem alternativas fornecidas pelo IBM Bluemix Private Cloud

      Nesta liberação, o IBM Bluemix Private Cloud é fornecido previamente preenchido com imagens x86_64 do Cirros 0.3.3. Há imagens de nuvem adicionais disponíveis, fornecidas pelo IBM Bluemix Private Cloud como cortesia para o cliente.

      É possível fazer upload e instalar um software do sistema operacional adquirido separadamente ou é possível fornecer qualquer uma das imagens de sistema operacional a seguir que são disponibilizadas para você como uma instância de máquina virtual. Você é responsável por obedecer a todos os termos da licença do sistema operacional e por adquirir autorizações adequadas para cada instância da máquina virtual.

      • CentOS 6.8 x86_64
      • CentOS 7.2 x86_64
      • Ubuntu Server 12.04 LTS x86_64
      • Ubuntu Server 14.04 LTS x86_64
      • Windows Server 2008 R2 SP1 Datacenter
      • Windows Server 2008 R2 SP1 Enterprise
      • Windows Server 2008 R2 SP1 Standard
      • Windows Server 2012 R2 Datacenter
      • Windows Server 2012 R2 Standard

      Para qualquer instância de cálculo virtual do sistema operacional Microsoft Windows, estes são os fornecimentos de cliente específicos: o cliente precisa ter e manter seu próprio servidor de licença do Microsoft Windows para a ativação dessas instâncias de cálculo virtual do sistema operacional Windows. Consulte Como ativar licenças do Windows no IBM Bluemix Private Cloud para obter mais informações.

      Obter downloads de imagens de nuvem do Box Panel

      1. Efetue login no Box Panel com as credenciais do Box Panel. A página Imagens da nuvem no Box Panel fica disponível se houver pelo menos uma nuvem e se você for um contato de cliente Primário ou Técnico.

      2. No painel do Box Panel, clique em Serviços e selecione Imagens de nuvem.

      3. A página Imagens da nuvem será exibida. Cada imagem é exibida como uma placa.

      4. Clique na placa que exibe a imagem de seu interesse. Será exibida uma janela modal, contendo a tempURL.

      5. A tempURL exibida será válida pelas próximas 24 horas, para que seja possível fazer o download da imagem de nuvem.

      Fazer upload de uma imagem para o repositório Glance do IBM Bluemix Private Cloud utilizando a tempURL da imagem da nuvem

      Notas:

      • Somente os usuários que têm a função cloud_admin poderão tornar uma imagem pública para outros usuários e para o projeto.
      • O upload de uma imagem para o Glance utilizando a tempUrl diretamente como um local de imagem não pode ser usado para o IBM Bluemix Private Cloud que bloqueou o acesso de Internet de saída do painel de controle do OpenStack.
      • O upload de uma imagem para o Glance utilizando a tempUrl diretamente como um local de imagem é suportado somente no IBM Bluemix Private Cloud 2.1.0 e posteriores e não em liberações ANTERIORES a 2.1.0.
      • Para IBM Bluemix Private Clouds em liberações ANTERIORES a 2.1 sem acesso de Internet de saída, siga as instruções em Fazer upload de uma imagem para um repositório Glance do IBM Bluemix Private Cloud usando arquivos de imagem transferidos por download.

      É possível fazer upload de uma imagem com o painel do OpenStack ou o cliente da linha de comando.

      Fazer upload de uma imagem usando o painel do OpenStack

      1. Efetue login no painel do IBM Bluemix Private Cloud OpenStack.
      2. No painel Projeto, expanda Calcular e clique em Imagens.

      3. Clique em Criar imagem na página. Especifique os parâmetros a seguir na nova página e clique em Criar imagem.

        Item Descrição
        Nome Insira um nome para a imagem.
        Descrição Opcionalmente, insira uma breve descrição da imagem.
        Origem da imagem Escolha Local da imagem.
        Local da Imagem Especifique a tempURL de download da imagem da nuvem.
        Tipo de Selecione o formato correto (por exemplo, QCOW2) para a imagem.
        arquitetura Especifique a arquitetura (por exemplo, i386 para uma arquitetura de 32 bits ou x86_64 para uma arquitetura de 64 bits).
        Disco Mínimo (GB) Para uma imagem do Linux, use 5; para uma imagem do Windows, use 30.
        RAM Mínima (MB) Para uma imagem do Linux, use 512; para uma imagem do Windows, use 2048
        Dados de Cópia Esta opção deve ser marcada para copiar os dados de imagem para o serviço de imagem.
        Público Selecione esta caixa de seleção para tornar a imagem pública para todos os usuários em todos os projetos.
        Protegido Selecione esta caixa de seleção para assegurar que apenas os usuários com permissões possam excluir a imagem.

        Por exemplo:

         Name: Ubuntu Server 14.04 LTS x86_64 - 20160301
 Description: Ubuntu Server 14.04 LTS x86_64 cloud image released on 2016/03/01
 Image Source: Image Location
 Image File: <tempURL_for_QCOW2>
 Format: QCOW2 - QEMU Emulator
 Architecture: x86_64
 Minimum Disk(GB): 5
 Minimum RAM(MB): 512
 Copy Data: True
 Public: True
      4. Verifique se a imagem criada aparece na lista de imagens. A imagem é enfileirada para upload. Pode demorar um pouco para que o status mude de “Enfileirado” para “Ativo”.

      Nota: somente os usuários que têm a função cloud_admin poderão tornar uma imagem pública para outros usuários e para o projeto.

      Fazer upload de uma imagem usando o cliente da linha de comando OpenStack

      É possível fazer upload de imagens por meio do cliente Glance. Consulte Instalando OpenStack Clients para obter mais informações.

      Crie uma imagem.

      • Para imagens do Linux, use -- min-disk 5 --min-ram 512

      • Para imagens do Windows, use --min-disk 30 --min-ram 2048

        $ glance --os-image-api-version 1 image-create --name <image_name> --min-disk <min-disk> --min-ram <min-ram> --disk-format qcow2 --container-format bare --is-public True --copy-from  <tempURL_to_qcow2_image_file>

      Verifique se a imagem foi criada com êxito. A imagem é enfileirada para upload. Pode demorar um pouco para que o status mude de “Enfileirado” para “Ativo”.

      $ glance image-show <image-id>

      Fazer upload de uma imagem para um repositório Glance do IBM Bluemix Private Cloud usando arquivos de imagem transferidos por download

      Notas:

      • Somente os usuários que têm a função cloud_admin poderão tornar uma imagem pública para outros usuários e para o projeto.

      • Ao fazer download da imagem da nuvem usando a tempUrl certifique-se de que o token da tempUrl tenha caracteres de escape ou seja especificado entre aspas. Por exemplo:

        curl -f -o <output_file> '<cloud_image_download_tempUrl>'  
curl -f -o ubuntu-guest-image-14.04-20160301-x86_64.qcow2 'https://dal05.objectstorage.softlayer.net/v1/AUTH_2201d7be-5d96-431c-9bd0-ec3ed5b62b19/cloud_images/test/ubuntu-guest-image-14.04-20160301-x86_64.qcow2?temp_url_sig=9e9678ad2d81489cebd032dff2332ec8ee50ebba&temp_url_expires=1458956867'

      Siga as seguintes etapas:

      1. Faça download dos arquivos .qcow2 e .md5sum da imagem.

      2. Faça download do arquivo de imagem .qcow2 e do arquivo de soma de verificação .md5sum usando a tempURL e coloque esses dois arquivos na mesma pasta. Por exemplo:

         ubuntu-guest-image-14.04-20160301-x86_64.qcow2
 ubuntu-guest-image-14.04-20160301-x86_64.md5sum

      3. Verifique o arquivo de imagem .qcow2 transferido por download utilizando o arquivo de soma de verificação .md5sum. Por exemplo:

         md5sum -c ubuntu-guest-image-14.04-20160301-x86_64.md5sum
 ubuntu-guest-image-14.04-20160301-x86_64.qcow2: OK

        Se o resultado retornar “OK”, a imagem qcow2 transferida por download é válida.

      4. Faça upload da imagem para o Glance. É possível fazer upload da imagem com o painel do OpenStack ou com o cliente da linha de comando.

      Conectando à sua instância

      Nota: certifique-se de que as regras de segurança adequadas estejam definidas para a conexão da instância por meio de ssh ou rdp.

      1. Efetue login no painel do IBM Bluemix Private Cloud OpenStack.

      2. Expanda o painel Projeto, em seguida, a guia Calcular na área de navegação esquerda e clique em Instâncias.

      3. Localize sua instância na lista de instâncias em execução e anote seu endereço IP, como 192.168.100.237.

      4. Crie uma conexão com a instância, especificado seu endereço IP.

      Conectando-se a instâncias do Linux

      Conecte-se à instância por meio de SSH ou VNC. O userid padrão para as imagens do Linux fornecidas pela IBM é ibmcloud. Após a conexão com a instância do Linux, será possível usar o comando sudo para executar os comandos que geralmente requerem acesso raiz. O Sudo é configurado para não solicitar a senha para o userid padrão.

      • Use o cliente SSH de sua preferência e abra uma conexão com o endereço IP correto da instância de MV, usando a chave SSH privada do par de chaves especificado durante o fornecimento:

          ssh -i <path_of_Your_SSH_private_key_file> <userid>@<ip_of_instance>

      • Visualize o console do VNC da instância, clicando no nome da instância no painel do OpenStack do IBM Bluemix Private Cloud e, em seguida, clique na guia Console. O console VNC se conecta com HTTPS.

      Nota: para poder efetuar login em uma nova instância do Linux por meio do console como o usuário ibmcloud, você deve ter especificado uma senha no Script de customização durante o fornecimento.

      Conectando-se a instâncias do Windows

      Conecte-se à instância por meio da Área de trabalho remota ou de VNC. Será necessário usar o userid e a password especificados no Script de customização durante o fornecimento da MV. No primeiro login, é exibido um aviso, solicitando a mudança da senha. Faça isso.

      • Use seu cliente Remote Desktop favorito para se conectar.
      • Visualize o console do VNC da instância, clicando no nome da instância no painel do OpenStack do IBM Bluemix Private Cloud e, em seguida, clique na guia console. O console VNC se conecta com HTTPS.

      Atualização de imagem

      Ao receber uma notificação das operações do IBM Bluemix Private Cloud informando que as imagens da nuvem foram atualizadas, é possível desativar a imagem existente e recriar uma nova imagem usando a URL de download obtida no Box Panel para a imagem atualizada.

      Usando o cliente da linha de comando do OpenStack para desativar a imagem existente

      1. Obtenha o ID da imagem existente:

         $ glance image-list

      2. Desative a imagem:

        $ glance image-deactivate <image_id>

      3. Mostre o status da imagem:

        $ glance image-show <image_id>

      4. Depois de desativar a imagem existente, recrie a nova imagem, seguindo as etapas em Fazer upload de uma imagem para o repositório Glance do IBM Bluemix Private Cloud utilizando a tempURL da imagem da nuvem neste guia.

        Exemplo: ubuntu-guest-image-14.04-20160301-x86_64.qcow2
