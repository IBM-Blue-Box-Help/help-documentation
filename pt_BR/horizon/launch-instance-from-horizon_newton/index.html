      Launching an instance from the Horizon dashboard (OpenStack Newton release)

      • by Ying Tang ● March 31, 2017

      • Tags:

        • horizon

        • instance

        • launch

        • newton

        • rhosp

      Etapas do SQL

      1. Efetue login no painel do Horizon.
      2. No tópico Projeto, expanda Cálculo e clique em Instâncias. Caso haja instâncias existentes, é possível visualizar uma lista das instâncias com seus detalhes, como nome, endereço IP e status. Para obter informações sobre como gerenciar as instâncias, consulte Como gerenciar as instâncias.
      3. Clique em Ativar instância na parte superior direita do painel do Horizon. O diálogo Ativar instância é exibido.

      Navegue nas guias a seguir e forneça os valores, em seguida, clique em Ativar instância na parte inferior direita da caixa de diálogo.

      Preencha valores na caixa de diálogo Ativar instância

      A guia Detalhes

      Valores Descrição
      Nome da Instância Designe um nome para a máquina virtual. O nome designado aqui se torna o nome do host inicial do servidor. Não há garantia de exclusividade ao criar os nomes dos servidores, portanto, é possível que haja duas instâncias com o mesmo nome de host.
      Zona de Disponibilidade Por padrão, esse valor é configurado como a Zona de disponibilidade (AZ) fornecida pelo provedor de nuvem, por exemplo, compute_standard.
      Contagem Para disparar diversas instâncias, insira um valor maior que 1. O padrão é 1.

      Para continuar, clique em Avançar ou clique na guia Origem.

      A guia Origem

      O campo Selecionar origem de inicialização oferece estas opções:

      Opções Descrição
      Imagem Selecione a partir da lista de imagens privadas e públicas na tabela Disponível, em seguida, clique no sinal de mais (+) para movê-la para Alocado. Opcionalmente, é possível clicar em Sim em Criar novo volume para ativar mais opções de criação de volumes. Caso você tenha criado um novo volume com essa instância, é possível clicar em Sim na opção Excluir volume ao excluir a instância se quiser excluir esse volume ao finalizar a instância.
      Captura Instantânea da Instância Selecione a partir das capturas instantâneas na tabela Disponível e clique no sinal de mais (+) para movê-las para Alocado.
      Volume Selecione a partir dos volumes na tabela Disponível, em seguida, clique no sinal de mais (+) para movê-lo para Alocado. Opcionalmente, é possível clicar em Sim na opção Excluir volume ao excluir a instância para excluir esse volume ao finalizar a instância.
      Captura Instantânea do Volume Selecione a partir das capturas instantâneas de volume na tabela Disponível, em seguida, clique no sinal de mais (+) para movê-las para Alocado.Opcionalmente, é possível clicar em Sim na opção Excluir volume ao excluir a instância para excluir esse volume ao finalizar a instância.

      Nota: a opção Imagem é configurada por padrão, portanto, não é necessário selecionar uma imagem antes da inicialização. Ao usar a opção Imagem, é possível selecionar a partir das imagens públicas fornecidas pela IBM, ou é possível fazer upload e usar sua própria imagem. Para obter informações sobre como fazer upload de uma imagem para o IBM Bluemix Private Cloud, consulte Imagens de nuvem fornecidas pelo IBM Bluemix Private Cloud.

      A guia Tipo

      Selecione a partir da lista de tipos, em seguida, clique no sinal de mais (+) para movê-lo para Alocado.

      A guia Redes

      Selecione a partir da lista de redes, em seguida, clique no sinal de mais (+) para movê-la para Alocado. É possível alocar várias redes para a instância.

      A guia Portas de rede

      Se nenhuma porta de rede for especificada, uma porta padrão será designada para a instância quando a instância for criada e inicializada.

      Ocasionalmente, pode ser necessário criar as portas antecipadamente por meio de APIs. Depois de criar uma porta e associá-la a uma das redes selecionadas na guia Redes, será possível visualizar essa porta na tabela Disponível. É possível clicar no sinal de mais (+) para movê-la para Alocado.

      A guia Grupos de segurança

      Selecione a partir da lista de grupos de segurança, em seguida, clique no sinal de mais (+) para movê-lo para Alocado.

      A guia Par de chaves

      Especifique um par de chaves. Você tem as seguintes opções:

      • Selecione a partir da lista de pares de chaves armazenados, em seguida, clique no sinal de mais (+) para mover a seleção para Alocado.
      • Clique em Importar par de chaves para importar seu próprio par de chaves. Copie e cole a chave pública e, em seguida, clique em Importar par de chaves.

      • Clique em Criar par de chaves para criar e armazenar um novo par de chaves. Especifique o nome do par de chaves, em seguida, clique em Criar par de chaves.

        É possível salvar o novo par de chaves no host local.

      A Guia Configuração

      Os scripts customizados anexados às instâncias podem executar ações específicas quando a instância é ativada. É possível digitar o script diretamente no campo Script de customização.

      Para instâncias do Linux: o nome do usuário padrão é ibmcloud. Para ativar a autenticação de senha por meio do console e de SSH, use os dados de script a seguir para o Script de customização (substitua o campo password relevante por <YOUR_PASSWORD>):

      #cloud-config password: <YOUR_PASSWORD> chpasswd: { expire: False } ssh_pwauth: True

      Para instâncias do Windows: para configurar o nome de usuário e a senha iniciais, use os dados de script a seguir para o Script de customização (substitua os campos username e password relevantes por <YOUR_USERNAME> e <YOUR_PASSWORD>).

      rem cmd
net user <YOUR_USERNAME> <YOUR_PASSWORD> /logonpasswordchg:yes /add /y
net localgroup administrators <YOUR_USERNAME> /add

      Caso o navegador suporte a API de arquivo HTML5, é possível clicar em Procurar para carregar o script a partir de um arquivo. O tamanho do script não deve exceder 16 Kb.

      No menu Partição de disco, é possível selecionar Automático para redimensionar o disco e configurá-lo para uma única partição ou é possível selecionar Manual para criar várias partições no disco.

      Marque a caixa de seleção Unidade de configuração se quiser gravar metadados em uma unidade de configuração especial, em seguida, especifique os metadados na guia Metadados. Ao ser inicializada, a instância é anexada à unidade de configuração para acessar os metadados.

      A guia Grupos de servidores

      É possível usar grupos de servidores para controlar a política de planejamento de afinidade e antiafinidade para um grupo de servidores (instâncias).

      Ocasionalmente, pode ser necessário criar grupos de servidores antecipadamente por meio de APIs. Depois de criar um grupo de servidores, será possível visualizar esse grupo de servidores na tabela Disponível. É possível clicar no sinal de mais (+) para movê-la para Alocado.

      A guia Sugestões do planejador

      É possível usar as sugestões do planejador para transmitir informações adicionais relacionadas ao posicionamento para o planejador Nova. Por exemplo, é possível usar uma sugestão do planejador para assegurar que uma instância seja planejada para um host a partir de um determinado grupo de servidores. Também é possível construir uma sugestão de planejador customizado em formato JSON. Para obter mais detalhes sobre o mecanismo de planejamento e os tipos de sugestões suportadas, consulte Planejadores de cálculo.

      A guia Metadados

      O Glance fornece uma API comum para definir pares de chave e valor que identificam metadados para diferentes tipos de recursos (imagens, artefatos, volumes, tipos, agregados, etc.). Caso a imagem selecionada contenha metadados predefinidos, você verá isso na coluna Metadados disponíveis.

      É possível criar pares de chave e valor adicionais no campo Customizado e clicar no sinal de mais (+) para movê-los para a lista Metadados existentes. Por exemplo, metadados customizados é um parâmetro que você gostaria de transmitir para o aplicativo, portanto, ele pode ser instalado com a instância.

      Para obter mais informações sobre metadados do Glance, consulte Conceitos de definição de metadados e Utilizando as APIs públicas do catálogo de definições de metadados do Glance.

      Como se conectar à instância

      Nota: certifique-se de que as regras de grupo de segurança adequadas estejam definidas para a conexão da instância por meio de ssh ou rdp.

      1. Efetue login no painel do Horizon do IBM Bluemix Private Cloud.

      2. Expanda o painel Projeto, em seguida, a guia Calcular na área de navegação esquerda e clique em Instâncias.

      3. Localize sua instância na lista de instâncias em execução e anote seu endereço IP, como 192.168.100.237.

      4. Crie uma conexão com a instância, especificado seu endereço IP.

      Conectando-se a instâncias do Linux

      Conecte-se à instância por meio de SSH ou VNC. O userid padrão para as imagens do Linux fornecidas pela IBM é ibmcloud. Após a conexão com a instância do Linux, é possível usar o comando sudo para executar comandos que normalmente requerem acesso raiz. O Sudo é configurado para não solicitar a senha para o userid padrão.

      • Utilizando o cliente SSH de sua preferência, abra uma conexão com o endereço IP correto da instância de MV, usando a chave SSH privada do par de chaves especificado durante o fornecimento:

          ssh -i <path_of_Your_SSH_private_key_file> <userid>@<ip_of_instance>

      • No painel do IBM Bluemix Private Cloud Horizon, visualize o console VNC da instância, clicando em Console na lista suspensa na coluna Ações da instância. O console VNC se conecta com HTTPS.

      Nota: para efetuar login em uma nova instância do Linux por meio do console, como o usuário ibmcloud, você deve ter especificado uma senha anteriormente no Script de customização, durante o fornecimento.

      Conectando-se a instâncias do Windows

      Conecte-se à instância por meio da Área de trabalho remota ou de VNC. Será necessário usar o userid e a password especificados no Script de customização durante o fornecimento da MV. No primeiro login, é exibido um aviso, solicitando a mudança da senha. Faça isso.

      • Use seu cliente Remote Desktop favorito para se conectar.
      • No painel do IBM Bluemix Private Cloud Horizon, visualize o console VNC da instância, clicando em Console na lista suspensa na coluna Ações da instância. O console VNC se conecta com HTTPS.

      Como gerenciar as instâncias

      No tópico Projeto, expanda Cálculo e clique em Instâncias.

      O painel exibe a lista de instâncias para o projeto atual. Na coluna Ações, uma lista suspensa exibe as opções, como mostrado abaixo:

      • Criar captura instantânea: cria uma captura instantânea da instância.*
      • Associar IP flutuante: associa um IP flutuante disponível a uma porta da instância.
      • Anexar uma interface: anexa a interface à rede.
      • Remover uma interface: remove uma porta que foi criada para a instância.
      • Editar instância: muda o nome da instância ou faz upload dos grupos de segurança.
      • Atualizar metadados: atualiza os metadados customizados especificados para a instância.
      • Editar grupos de segurança: remove um grupo de segurança existente ou designa um novo grupo de segurança.
      • Console: visualiza o console VNC da instância.
      • Visualizar log: visualiza o log da instância.
      • Pausar instância: pausa a instância e salva seu estado na memória. Esta opção mantém os recursos (vCPUs e memória).
      • Suspender instância: suspende a instância e salva seu estado no disco. Esta opção libera recursos (vCPUs e memória).
      • Arquivar instância: arquiva a imagem e encerra a instância.
      • Redimensionar redimensionar: atualiza a configuração de tipo da instância.
      • Bloquear instância: bloqueia a instância.
      • Desbloquear instância: desbloqueia a instância.
      • Reinicialização soft da instância: executa um encerramento normal e reinicia a instância.
      • Reinicialização hard da instância: executa o equivalente a uma reconfiguração de energia do servidor.
      • Encerrar instância: desliga a instância.
      • Reconstruir instância: reconstrói a instância com outra imagem.
      • Excluir instância: exclui a instância.

      Clique na opção da operação a ser aplicada.
