      Image Configuration Parameters for RHOSP

      • by Leslie Lundquist ● March 28, 2017

      • Tags:

        • glance

        • userguides

        • image

        • glance

        • distro

        • libvirt

        • property keys

      Ao trabalhar com o IBM Bluemix Private Cloud com Red Hat, as seguintes chaves podem ser usadas com a opção de propriedade para os comandos glance image-update e glance image-create.

      $ glance image-update IMG-UUID --property architecture=x86_64

      Nota: O conjunto de comportamentos usando as propriedades da imagem substitui o conjunto de comportamentos usando tipos.

      Chaves da Propriedade

      Aplica-se a Chave Descrição Valores suportados
      Todos architecture A arquitetura da CPU que deve ser suportada pelo hypervisor. Por exemplo, x86_64 é o tipo de CPU atualmente suportado. Extensão de 64 bits x86_64 de IA-32
      Todos instance_uuid Para imagens de captura instantânea, este é o UUID do servidor usado para criar essa imagem. UUID do servidor válido
      Todos kernel_id O ID de uma imagem armazenada no Image Service que deve ser usada como o kernel ao inicializar uma imagem de estilo AMI. ID de imagem válido
      Todos os_distro O nome comum da distribuição do sistema operacional em minúsculas (usa o mesmo vocabulário de dados que o projeto libosinfo). Especifique apenas um valor reconhecido para este campo. Os valores são listados abaixo para ajudá-lo a procurar o valor reconhecido. (Veja a lista abaixo)
      Todos os_version A versão do sistema operacional conforme especificado pelo distribuidor. Número da versão (por exemplo, “11.10”)
      Todos ramdisk_id O ID da imagem armazenada no Image Service que deve ser usada como o disco virtual ao inicializar uma imagem de estilo AMI. ID de imagem válido
      Todos vm_mode O modo da máquina virtual. Isso representa a ABI (Interface Binária do Aplicativo) de host/guest usada para a máquina virtual. hvm-totalmente virtualizado. Este é o modo usado por QEMU e KVM.
      Driver de API libvirt hw_disk_bus Especifica o tipo de controlador de disco ao qual anexar os dispositivos de disco. scsi, virtio, ide ou usb.
      Driver de API libvirt hw_qemu_guest_agent Suporte de agente guest. Se configurado como sim, e se qemu-ga também for instalado, os sistemas de arquivos poderão estar em modo quiesce (congelado) e capturas instantâneas criadas automaticamente. sim / não
      Driver de API libvirt hw_rng_model Inclui um dispositivo gerador de números aleatórios nas instâncias da imagem. O administrador em nuvem pode ativar e controlar o comportamento do dispositivo configurando o tipo da instância. Por padrão: o dispositivo gerador é desativado, e /dev/random é usado como a origem de entropia padrão. Para especificar um dispositivo HW RNG físico, use a seguinte opção no arquivo nova.conf: rng_dev_path=/dev/hwrng virtio, ou outro dispositivo suportado.
      Driver de API libvirt hw_scsi_model Permite o uso de VirtIO SCSI (virtio-scsi) para fornecer acesso ao dispositivo de bloco para instâncias de cálculo; por padrão, as instâncias usam o VirtIO Block (virtio-blk). VirtIO SCSI é um dispositivo do controlador SCSI para-virtualizado que fornece melhor escalabilidade e desempenho, e suporta hardware SCSI avançado. virtio-scsi
      Driver de API libvirt hw_video_model O driver de imagem de vídeo usado. vga, cirrus, vmvga, xen ou qxl
      Driver de API libvirt hw_video_ram Máximo de RAM para a imagem de vídeo. Usado apenas se um valor hw_video:ram_max_mb tiver sido configurado em extra_specs do tipo e esse valor for maior que o valor configurado em hw_video_ram. Número inteiro em MB (por exemplo, ‘64’)
      Driver de API libvirt hw_watchdog_action Ativa um dispositivo watchdog de hardware virtual que executa a ação especificada se o servidor for interrompido. O watchdog usa o dispositivo i6300esb (emulando um PCI Intel 6300ESB). Se hw_watchdog_action não for especificado, o watchdog será disabled. disabled -O dispositivo não está anexado. Permite que o usuário desative o watchdog para a imagem, mesmo se ele tiver sido ativado usando o tipo da imagem. O valor padrão para este parâmetro é disabled. reset-Forçosamente reconfigura o guest. poweroff-Forçosamente desliga o guest. pause-Pausa o guest. none-Ative apenas o watchdog; não faça nada se o servidor for interrompido.
      Driver de API libvirt os_command_line A linha de comandos do kernel a ser usada pelo driver libvirt, em vez do padrão. Para Linux Containers (LXC), o valor é usado como argumentos para a inicialização. Essa chave é válida apenas para o kernel Amazon, ramdisk ou imagens da máquina (aki, ari ou ami).  
      Driver de API libvirt hw_vif_model Especifica o modelo do dispositivo da interface de rede virtual a ser usado. As opções válidas dependem do hypervisor configurado. KVM e QEMU: e1000, ne2k_pci, pcnet, rtl8139 e virtio.
